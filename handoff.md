## 이번에 한 일 (완료)

- **TQQQ 추가**: 웹 화면에 TQQQ 카드(현재가·등락률·고점대비 하락률) 추가.

  - 코스피·코스닥: -30/-40/-50/-60/-70% 단계별 알람 (S&P500 동반 하락 시 미국 배분 안내)
  - TQQQ: 1차(-10%~-49%, 40회), 2차(-50% 아래 가격 10등분)
  - 신고점 리셋, -50% 재진입 시 기준가 재설정, 중복 알람 방지까지 포함
- **알람시계 확장**: `scheduler.py` 가 평일 15:30 브리핑에 더해, **10분마다 알람 조건을 검사**합니다.
- **웹 화면**: "🔔 투자 타이밍 현황" 섹션 추가 (`/api/alerts` 주소).
- **백테스트**: `backtest.py` 로 과거 데이터 시뮬레이션 가능.
- **시장 데이터 정합성 개선 (최근 업데이트)**:
  - **장중 최고가(High) 기반 ATH 계산**: 기존 일봉 종가(`Close`) 기준에서 실제 차트 고점과 일치하는 **'장중 최고가(High)'** 기준으로 계산하도록 수정하여 KOSPI 전고점을 `9,385.59`로 일치시켰습니다.
  - **TQQQ 분할 조정 ATH 수정**: TQQQ의 전고점 기본값을 분할 조정 기준 실제 장중 고점인 **`87.89`**로 수정했습니다.
  - **S&P 500 스케일링 계수 수정**: KIS API에서 SPY ETF로 지수를 환산할 때 계수를 `10.05`배로 적용하여 S&P 500 index 실시간 지수와 오차를 거의 제로 수준으로 맞췄습니다.
  - **코스닥 지수 코드 수정**: KIS API에서 코스닥 코드 조회를 `2001`에서 실제 지수 코드인 `1001`로 수정하여 데이터 수집 오류를 해결했습니다.
  - **네이버 금융(Naver Finance) 실시간 연동**: 야후 파이낸스 차단 우회 및 한투 API의 15분 지연 시세를 극복하기 위해, 해외 주식(`S&P 500`, `TQQQ`) 조회 시 **네이버 금융의 실시간 시세 API를 1순위로 조회**하도록 보완했습니다. 이로 인해 TQQQ 실시간 시세(`74.44`) 및 ATH(`87.89`)가 완벽하게 일치하게 되었습니다.
  - **대시보드 UI 전고점 수치 표시**: 각 지수 카드 하단에 하락률 뿐만 아니라 `역대 최고가 (수치)`가 직접 괄호 안에 표시되도록 개선하여, 화면에서 즉시 확인 가능합니다.

## 사용자가 직접 해야 하는 일 (아직 안 됨)

봇이 아직 **텔레그램과 연결되지 않았습니다.** 아래 3가지만 하면 됩니다.

1. 텔레그램에서 **@BotFather** 에게 `/newbot` → 봇 만들고 **토큰** 받기
2. 만든 봇에게 아무 말이나 한 번 보내고, 브라우저에서
   `https://api.telegram.org/bot<토큰>/getUpdates` 열어서 `"chat":{"id": ...}` 숫자(= **방 번호**) 확인
3. `.env` 파일에 `TELEGRAM_BOT_TOKEN` 과 `TELEGRAM_CHAT_ID` 넣기
4. `python scheduler.py --test` 로 톡 오는지 확인 → 되면 `python scheduler.py` 로 상시 실행

## 서버 터미널에서의 실행 및 업데이트 가이드

가상환경(venv)이 비활성화되거나 수치가 맞지 않을 경우 다음 순서대로 실행하십시오.

```bash
# 1. 프로젝트 폴더 내 가상환경(venv) 활성화
source venv/bin/activate

# 2. 최신 코드 풀(Pull)
git reset --hard origin/master
git pull

# 3. 낡은 고점이 들어있는 데이터베이스 삭제 (매수 타이밍 캐시 꼬임 방지)
rm alert_state.db

# 4. 새로운 고점 기준으로 알림 조건 즉시 갱신 (코스닥 -30% 이하 1차 매수 및 TQQQ -15% 등 반영)
python scheduler.py --check-alerts

# 5. 테스트 캡처 실행하여 텔레그램으로 대시보드 스크린샷 전송
python scheduler.py --test
```

## "지금 당장 보여줘" 기능 (완료)

정해진 시간이 아니어도 **명령/호출 한 번에 즉시 반응**하는 기능을 붙였습니다.
오픈클로(봇 2)가 이 주소들을 불러서 대화형으로 답할 수 있습니다.

- `GET /api/summary` → 현재 시장·알람 상황을 **글자로 요약** ("지금 시장 어때?")
- `GET /api/screenshot` → 대시보드를 **즉시 캡처한 이미지**로 반환 ("화면 보여줘")
- `POST /api/briefing/send` → 지금 즉시 캡처해서 **봇 1로 전송**
- 터미널: `python scheduler.py --summary` (요약 출력)

## 봇 2개 분리 (완료된 설계 / 사용자 설정 대기)

- **봇 1 (브리핑 봇)**: 우리 코드가 자동 전송. 토큰 → `.env`의 `BRIEFING_BOT_TOKEN` (없으면 기존 `TELEGRAM_BOT_TOKEN`으로 대체 → 서버의 기존 토큰과 안 섞임)
- **봇 2 (오픈클로 봇)**: 오픈클로가 관리. 토큰 → 오픈클로 설정 (우리 `.env` 아님)
- ⚠️ 봇 1·봇 2 토큰을 섞으면 충돌합니다. 반드시 2개로 나누세요.
- 정확한 숫자 알람은 우리 코드가, 대화형 응답은 오픈클로가 담당하는 역할 분리 구조.
- 자세한 계획·설정 순서: `docs/telegram-two-bot-plan.md`

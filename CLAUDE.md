> 이 파일은 Claude(그리고 사람)가 이 프로젝트를 빠르게 이해하도록 돕는 안내서입니다.
> **설명은 항상 쉽게, 어려운 용어는 풀어서** 해주세요. (예: "엔드포인트" → "주소", "트리거" → "조건 알람")

## 이 프로젝트가 하는 일 (한 줄)

한국투자증권 API로 코스피·코스닥·S&P500·TQQQ 시세를 가져와, 예쁜 웹 화면에 보여주고, 매일 정해진 시간과 **하락 알람 조건**에 맞춰 텔레그램으로 톡을 보내주는 봇입니다.

## 큰 그림 (동작 순서)

1. `market_data.py` 가 시세를 모아옵니다. (한투 API 먼저, 안 되면 야후 파이낸스로 대신)
2. 웹 화면(`app.py` + `templates/index.html`)이 그 시세를 카드로 보여줍니다.
3. `trigger_engine.py` 가 "지금 고점 대비 몇 % 빠졌나?"를 계산해서 살 타이밍인지 판단합니다.
4. `scheduler.py` 가 시간을 지키는 알람시계입니다.
   - 평일 오후 3시 30분 → 화면을 사진으로 찍어(`capture.py`) 텔레그램 전송
   - 10분마다 → 새로 도달한 하락 단계가 있으면 텔레그램으로 알람
5. `notifier.py` 가 실제로 텔레그램에 글/사진을 보냅니다.

## 파일별 역할 (쉽게)

| 파일 | 하는 일 |
|------|---------|
| `app.py` | 웹 화면과 데이터 주소(`/api/indices`, `/api/alerts`)를 여는 서버 |
| `kis_client.py` | 한국투자증권 API 로그인·시세 요청 담당 |
| `market_data.py` | 시세 모아오기 + 역대 최고가 기억 (공용 창고) |
| `trigger_engine.py` | "지금 사야 할 타이밍인가?" 계산 + 중복 알람 방지 |
| `notifier.py` | 텔레그램에 글/사진 보내기 |
| `capture.py` | 웹 화면을 사진으로 찍기 |
| `scheduler.py` | 정해진 시간·주기에 위 작업들을 실행하는 알람시계 |
| `backtest.py` | 과거 데이터로 "그때였다면 알람이 언제 울렸을까" 연습 |
| `templates/index.html`, `static/style.css` | 웹 화면 디자인 |

## 알람 규칙 (돈이 걸린 부분 — 함부로 바꾸지 말 것)

- 기준은 항상 **역대 최고가 대비 하락률**. 신고점을 새로 찍으면 단계가 리셋됩니다.
- **코스피·코스닥**: 각각 따로 -30/-40/-50/-60/-70%에서 알람. 단계마다 시드 20%.
  - S&P500이 역대 최고가 대비 -10% 넘게 같이 빠지는 중이면 → "미국 시장에 넣으라"고 안내.
- **TQQQ 1차**: -10%부터 1%씩 더 빠질 때마다 1번씩, 총 40번 (-10% ~ -49%).
- **TQQQ 2차**: -50% 찍은 순간의 가격 P를 기준으로 0~P를 10등분한 가격을 밑돌 때마다 매수.
- 같은 단계에서 두 번 울리지 않게 `alert_state.db`(작은 저장 파일)에 기록합니다.

## 봇 2개 구조 (역할 분리)

- **봇 1 (브리핑 봇)**: 우리 코드(`scheduler.py`)가 정해진 시간·조건에 자동으로 톡 전송. 토큰은 `.env`의 `TELEGRAM_BOT_TOKEN`.
- **봇 2 (오픈클로 봇)**: 오픈클로가 관리. 내가 물어보면 우리 프로그램의 주소를 불러서 답함. 토큰은 오픈클로 설정에 넣음 (우리 `.env`에는 안 넣음).
- 자세한 계획: `docs/telegram-two-bot-plan.md`

## 온디맨드 주소 (오픈클로가 부르는 "지금 당장" 기능)

| 주소 | 하는 일 |
|------|---------|
| `GET /api/summary` | 현재 시장·알람 상황을 글자로 요약 (`text` 필드) |
| `GET /api/screenshot` | 지금 즉시 캡처한 대시보드 PNG 이미지 반환 |
| `POST /api/briefing/send` | 지금 즉시 캡처해서 봇 1로 텔레그램 전송 |

## 자주 쓰는 명령어

```bash
# 지금 바로 화면 캡처해서 텔레그램 전송 (연결 테스트)
python scheduler.py --test

# 지금 바로 알람 조건 검사 (도달한 단계 있으면 톡 전송)
python scheduler.py --check-alerts

# 현재 상황을 글자로 요약해서 출력
python scheduler.py --summary

# 상시 실행 (평일 15:30 브리핑 + 10분마다 알람 검사)
python scheduler.py

# 과거 하락장으로 연습해 보기
python backtest.py --start 2021-11-01 --end 2023-01-01
```

## 설정값 (.env 파일)

- `BRIEFING_BOT_TOKEN`, `BRIEFING_CHAT_ID` : **봇 1(브리핑 봇)** 열쇠와 받을 방 번호 (권장). 없으면 `TELEGRAM_BOT_TOKEN`/`TELEGRAM_CHAT_ID`로 자동 대체 → 서버의 기존 토큰과 안 섞임
- `KIS_APP_KEY`, `KIS_APP_SECRET`, `KIS_ACCOUNT_NO`, `KIS_CANV_MODE` : 한국투자증권 API 설정
- `ALERT_CHECK_INTERVAL_MIN` : 알람 검사 주기(분), 기본 10분
- `CHROMIUM_PATH` : (선택) 크롬 실행 파일 경로 직접 지정. 미설정 시 Playwright 기본값 사용

## 규칙 / 주의

- `.env`, `.token_cache.json`, `alert_state.db` 는 **깃에 올리지 않습니다** (비밀·상태 파일).

# 📨 텔레그램 봇 2개 분리 계획서

> **목표**: 텔레그램 봇을 역할별로 2개로 나눈다.
> **최종 수정**: 2026-07-16

## 왜 2개로 나누나요? (쉽게)

봇 하나로 "자동 알림"과 "대화형 응답"을 같이 하면 서로 부딪힙니다.
(둘 다 같은 봇으로 메시지를 받으려고 하면 충돌해요.)
그래서 역할을 딱 나눕니다.

| 봇 | 이름(예시) | 하는 일 | 누가 보내나 |
|----|-----------|---------|-------------|
| **봇 1 — 브리핑 봇** | `MyMarketBriefingBot` | 정해진 시간·조건에 **자동으로** 톡 보냄 (15:30 마감 브리핑, 하락 알람) | 우리 프로그램(`scheduler.py`) |
| **봇 2 — 오픈클로 봇** | `MyMarketAssistantBot` | 내가 **물어보면 답함** ("지금 시장 어때?", "캡처 보여줘") | 오픈클로 |

- **봇 1**은 우리 코드가 직접 텔레그램으로 보냅니다. (지금 이미 만들어져 있음)
- **봇 2**는 오픈클로가 관리합니다. 오픈클로가 우리 프로그램의 "주소(API)"를 불러서 결과를 받아, 자기 봇으로 나에게 전달합니다.

```
[정기/조건 알림]   우리 프로그램(scheduler) ──보냄──▶ 봇 1 ──▶ 내 텔레그램
[물어보면 응답]    나 ──질문──▶ 봇 2 ──▶ 오픈클로 ──API 호출──▶ 우리 프로그램
                                          ◀──결과(글/그림)──
```

## 계획 1: 우리 프로그램에 "지금 당장" 기능 추가 (코드 작업 — 이번에 진행)

오픈클로(봇 2)가 불러 쓸 수 있도록, 정해진 시간이 아니어도 **바로 반응하는 주소**를 만듭니다.

| 주소(API) | 하는 일 | 오픈클로 활용 예 |
|-----------|---------|-----------------|
| `GET /api/summary` | 현재 시장·알람 상황을 **글자로 요약**해서 돌려줌 | "지금 시장 상황 알려줘" |
| `GET /api/screenshot` | 대시보드를 **사진으로 찍어** 이미지 파일로 돌려줌 | "지금 화면 캡처 보여줘" |
| `POST /api/briefing/send` | 지금 즉시 캡처해서 **봇 1로** 텔레그램 전송 | "지금 브리핑 한 번 쏴줘" |

명령줄(터미널)에서도 바로 쓸 수 있게:
```bash
python scheduler.py --summary        # 현재 상황을 글자로 출력
python scheduler.py --test           # 지금 캡처해서 봇 1로 전송
python scheduler.py --check-alerts   # 지금 알람 조건 검사
```

### 구현 단계
```
[x] 1단계: 현재 상황을 글자로 요약하는 함수 (summary.py)
[x] 2단계: 캡처만 하는 함수와 캡처+전송 함수 분리 (capture.py)
[x] 3단계: /api/summary, /api/screenshot, /api/briefing/send 주소 추가 (app.py)
[x] 4단계: python scheduler.py --summary 명령 추가
[ ] 5단계: (사용자) 봇 1 토큰을 .env에 넣고 python scheduler.py --test 로 확인
```

## 계획 2: 봇 설정 (사용자가 하는 일 — 텔레그램에서)

### A. 봇 2개 만들기
1. 텔레그램에서 **@BotFather** 검색 → `/newbot`
2. **봇 1**(브리핑용) 만들기 → 이름·아이디 정하고 → **토큰 A** 받아 적어두기
3. 다시 `/newbot` → **봇 2**(오픈클로용) 만들기 → **토큰 B** 받아 적어두기

### B. 내 방 번호(Chat ID) 확인
1. 봇 1에게 아무 말이나 한 번 보내기
2. 브라우저에서 `https://api.telegram.org/bot<토큰A>/getUpdates` 열기
3. `"chat":{"id": 12345678}` 처럼 나오는 숫자가 **내 방 번호(Chat ID)**

### C. 토큰 넣을 곳 (아주 중요 — 헷갈리지 마세요)
| 토큰 | 어디에 넣나 |
|------|-------------|
| **토큰 A** (봇 1) | 우리 프로그램의 `.env` 파일 → `BRIEFING_BOT_TOKEN` |
| **토큰 B** (봇 2) | **오픈클로 설정 화면** (우리 `.env`에는 넣지 않음) |

> 💡 서버 `.env`에 **이미 다른 봇 토큰(`TELEGRAM_BOT_TOKEN`)이 있어도 그대로 두세요.**
> 이 프로그램은 `BRIEFING_BOT_TOKEN`을 **먼저** 사용하고, 없을 때만 기존 `TELEGRAM_BOT_TOKEN`으로 대체합니다.
> 그래서 `BRIEFING_BOT_TOKEN`만 새로 채우면 기존 것과 **절대 안 섞입니다.**

`.env` 파일에 추가할 줄 (기존 내용은 건드리지 말고 아래만 추가):
```env
BRIEFING_BOT_TOKEN=토큰A_여기에
BRIEFING_CHAT_ID=내_방_번호_여기에
```
(전체 예시는 저장소의 `.env.example` 파일 참고)

### D. 확인
```bash
python scheduler.py --test   # 봇 1에서 사진 톡이 오면 성공
```

## ❓ "앱 웹을 캡처하려면 서버에서 앱을 켜야 하나요?" → 네, 맞습니다

캡처는 **실제 웹 화면을 브라우저로 열어 사진 찍는 방식**이라, 그 화면(웹 서버)이 켜져 있어야 합니다.
다만 따로 두 개를 켤 필요는 없습니다 — `scheduler.py`가 **웹 서버를 자동으로 같이 켭니다.**

| 하려는 것 | 웹 서버 필요? | 실행 방법 |
|-----------|:---:|-----------|
| 정기 브리핑 + 알람 (상시) | ✅ (자동으로 같이 켜짐) | `python scheduler.py` |
| 지금 캡처해서 전송 (테스트) | ✅ (자동으로 같이 켜짐) | `python scheduler.py --test` |
| 알람 조건만 검사 | ❌ (시세만 직접 조회) | `python scheduler.py --check-alerts` |
| 상황 글자 요약 | ❌ | `python scheduler.py --summary` |
| 오픈클로가 `/api/...` 호출 | ✅ (서버가 떠 있어야 함) | `python scheduler.py` 를 상시 실행 |

즉, 서버에서 `python scheduler.py`를 **상시 켜두면** → 자동 브리핑/알람도 되고, 오픈클로가 부를 주소도 열립니다.
(백그라운드 실행: `nohup python scheduler.py &` 또는 `pm2 start scheduler.py --name stock-bot`)

## 계획 3: 오픈클로 연결 (사용자가 하는 일 — 오픈클로에서)

1. 오픈클로에 텔레그램 채널 추가 → **토큰 B** 등록 → 페어링 승인
2. 오픈클로가 우리 프로그램 주소를 부를 수 있게, 프로그램을 켜 둠 (`python scheduler.py`)
   - 오픈클로가 같은 PC/서버에 있으면 `http://127.0.0.1:8000/api/summary` 같은 주소로 호출
   - 다른 곳에 있으면 외부에서 접근 가능한 주소가 필요 (예: 포트 열기 또는 터널)
3. 오픈클로에게 "시장 물어보면 `/api/summary` 불러서 알려줘" 같은 식으로 시켜두기
